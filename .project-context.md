# Project Context & Guidelines

## 1. Tech Stack (Vercel PWA)
- **Environment**: Progressive Web App (PWA) deployed via Vercel.
- **Strict Ban**: absolute ban on Capacitor (`npx cap sync`), native Android/iOS folders, and build commands.
- **Deployment**: GitHub Push -> Vercel Build -> Live Site.

## 2. Data Mapping Rules
- **Notion Rich Text**: ALWAYS use `rich_text[0]?.plain_text` to extract text. Never access `content` or other properties directly from the root of a rich_text object.

## 3. Work Process (Autonomous Protocol)
- **Silent Analysis**: Analyze impact on existing layout/logic BEFORE modification.
- **Atomic Fix**: Modify minimal code.
- **Self-Test**: Verify interactions (buttons, inputs, swipes) remain functional.
- **Reporting**: Report *Modification*, *Verification*, and *Operating Guarantee*.

## 4. Visual Verification (Mandatory)
- **Version Tag**: Update the visible version tag (e.g., `v1.0.1`) and rotate its background color (e.g., Red -> Blue -> Green) on EVERY user-facing change.
- **Purpose**: This provides immediate physical proof that the latest code is running on the mobile device.

## 5. UI/UX Standards
- **Font**: Pretendard (primary), Serif (secondary/accent).
- **Interactions**: Mobile-first touch handling (`touch-action: pan-y`, `dragPropagation`) is critical.
